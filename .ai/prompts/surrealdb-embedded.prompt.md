el proyecto actual quería disponer de la posibilidad de usar surrealdb sin depender del servidor, pero el SDK de golang no implementaba esta opción. He creado una librería golang **surrealdb-embedded** que implementa esta característica usando una shared library en Rust que sí implementa esta opción. La librería está en `/www/MCP/Remembrances/surrealdb-embedded`. El proyecto actual ya incorporaba variables de entorno para poder utilizar surrealdb indicando el path de la bbdd kv en formato rocksdb, reutiliza si sigue existiendo en el código este parámetro db-path, si recibe este parámetro prioriza en lugar de usar surrealdb remoto, utilizar esta librería para implementar toda la funcionalidad de persistencia en lugar de usar surrealdb con un servidor externo.

Anteriormente ya se han incluido en el proceso de compilación, shared libs de llama.cpp para la gestión de embeddings también de forma nativa. Utiliza el makefile si es necesario para la compilación con las shared libs si son necesarias en esta nueva librería que he creado surrealdb-embedded.
